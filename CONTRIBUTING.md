# Contributing to MTMS-ENGINE

Thank you for considering contributing to the MTMS-ENGINE project! We welcome contributions from the community. Here are some guidelines to ensure a smooth contribution process.

## Forking the Repository
1. Navigate to the [MTMS-ENGINE repository](https://github.com/tahirmalik6786/MTMS-ENGINE).
2. Click on the **Fork** button in the top-right corner.
3. This will create a copy of the repository in your own GitHub account.

## Branch Naming
When creating a new branch for your feature or bug fix, please follow these naming conventions:
- Use `feature/` for new features (e.g., `feature/add-login-page`)
- Use `bugfix/` for bug fixes (e.g., `bugfix/fix-typo`)
- Use `hotfix/` for urgent fixes (e.g., `hotfix/fix-crash-on-startup`)

## Commit Message Format
Please use the following format for your commit messages:

```
<type>(<scope>): <subject>

<body>

<footer>
```

- **type**: The type of the change (e.g., feat, fix, docs, style, refactor, perf, test)
- **scope**: The area of the code that the change relates to (optional)
- **subject**: A short description of the change (capitals, no period at the end)
- **body**: (optional) A more detailed description of the change
- **footer**: (optional) Any issues that are closed (e.g., `Closes #123`)

## Testing Requirements
- Ensure that your code passes all existing tests.
- Add new tests for any new features or bug fixes.
- You can run the tests using the command:
  
  ```
  npm test
  ```

## Pull Request Process
1. Ensure that your branch is up to date with the `main` branch before opening a pull request.
2. Click on the **Pull requests** tab of the original repository.
3. Click on **New pull request**.
4. Select your branch and submit the pull request.
5. Provide a clear description of your changes and mention any related issues.
6. Be open to feedback and make necessary changes based on that feedback.

Thank you for your contributions! We're excited to see what you will bring to the project!
